Неделя 7 визуализация¶

Определитесь, в каком виде вы хотите представить ваш проект. Мы предлагаем вам сделать интерактивное демо, содержащее:

  1. карты с визуализацией реального и прогнозируемого спроса на такси в выбираемый пользователем момент времени
  2. временной ряд фактического и прогнозируемого спроса на такси в выбираемой области.

Подход¶

Я пользуюсь библиотекой altair, которая построена на базе vega/vega-light, которая в свою очередь построена на d3. d3 это одная из самых развитых библиотек визуализации в javascript мире, а вот vega это относительно молодой проект который достаточно быстро развивается и у них главная фишка это дескриптивный язык описания интерактивной графики. Картография не является для них приоритетом, но наконец ее частично дотянули до altair, хоть как-то внятно работать она начала только в 2.0.0rc2, которая вышла нескольо дней назад. Может не все получиться, но хоть в радикально перемотренном API разберусь.

def display_row(*objects,**options):
    class MultiObectRow(object):
        def _repr_html_(self):
            r =""
            format_str = ''.join(['{}:{};'.format(a,b) for a,b in options.items()])
            for obj in objects:
                if hasattr(obj,"_repr_html_"):
                    obj_str =obj._repr_html_()
                else: obj_str=str(obj)
                r+='<div class="col" style="{}">{}</div>'.format(format_str,obj_str)
            return '<div class="row" style="display: flex;">{}</div>'.format(r)
    return display(MultiObectRow())

def _ddf(df_data,caption = ""): # чуть больше информации о pd.DF
    display (HTML("<h4>{2}</h4><p>({0}, {1})</p>".format(df_data.shape[0],df_data.shape[1],caption)))
    #.cm-s-ipython span.cm-variable
    if df_data.shape[1]>10:
        #display (HTML('<p class="code">{}</p>'.format(str(df_data.columns).replace('\n     ',''))))
        display (HTML('<p style="font-family: monospace;">{}</p>'.format(str(df_data.columns).replace('\n     ',''))))

    if df_data.shape[1]<4:
        display_row(df_data.head(5),df_data.sample(5).sort_index(),df_data.tail(5),margin ="1px 5px")
    elif(df_data.shape[1]<6):
        display_row(df_data.head(5),df_data.sample(5).sort_index(),margin ="1px 5px")
    elif(df_data.shape[1]<10):
        display(df_data.head(1).append(df_data.sample(5).sort_index()).append(df_data.tail(1)))

        display(df_data.head(1).append(df_data.sample(3).sort_index()).append(df_data.tail(1)).T)

Мы планируем показывать на карте количество вызовов, можно конечно показать красиву ровную шкалу цвета, но класики вызуалиазации не рекомендуют так делать ибо глаз достаточно полохо считывает токие разницы в цвете на карте. Рекомендуеться бинить на пять-семь групп и показывать уже цвета группы. Обычно и в нашем случае бинить даннные линейно достаточно бессмыслено, получится одна - две яркие точки на карте, а остальное будет однинакого цвета. Достаточно стандартное решение логарифмировать данные, чтобы как-то растянуть градации. Чтобы не грузить этими вычислениями javascript подготовим такую разбивку данных.
